Taak Repository van Rron Toska

• Structuur : hoe gestructureerd is de repository?

Er is structuur aanwezig. De gebruiker gebruikt mappen om zijn oefeningen te ordenen.

• Volledigheid : hoe veel oefeningen/opdrachten heeft de student toegevoegd

Er zijn een aantal oefeningen aanwezig. De onderwerpen die we in de les te zien krijgen past hij toe aan zijn persoonlijke website. Er is een map met oefeningen in verband met het interview zoals tabellen en lijsten. Ook heeft hij een map met daarin de website voor zijn klant.

• Regelmatigheid : hoe regelmatig plaatst de student updates (commits) in zijn repository? Dit kan je letterlijk uit de repository gegevens halen.

Er zijn twaalf commits op twee verschillende dagen geplaatst. De gebruiker upload een som van bestanden in één keer op.

• Correctheid : valideert de code?

In de code voor de website van de klant zijn er geen validatie fouten. In zijn persoonlijke site waarin hij oefeningen vanuit de les toepast zijn er wel fouten aanwezig, zoals het gebruik van center.

• Bereidheid tot verbetering: als je de commits bekijkt, zie je dan verbeteringen aan bestaande inhoud, of enkel nieuwe inhoud?

Het meerderheid van zijn inhoud is nieuw. De gebruiker wijzigt zijn code niet. In plaats daarvan verwijdert hij de inhoud dat fout is en upload hij een verbeterde versie op.
