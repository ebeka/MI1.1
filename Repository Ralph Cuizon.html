Taak Repository van Ralph Cuizon

• Structuur : hoe gestructureerd is de repository?

Er is een zeer goeie structuur aanwezig in de repository. Mijn medestudent gebruikt mappen om zijn oefeningen te ordenen namelijk 3 mappen: img, klant en oefeningen.

• Volledigheid : hoe veel oefeningen/opdrachten heeft de student toegevoegd

Mijn medestudent heeft al wat oefeninggen gemaakt. Ik zie hier vooral oefeningen die dateren van de eerste 4 lessen die we gehad hebben. ook heeft hij al enkele oefeningen gemaakt voor zijn klant. Daarnaast heeft hij een map met daarin de afbeeldingen die hij gebruikt voor zijn oefeningen.

• Regelmatigheid : hoe regelmatig plaatst de student updates (commits) in zijn repository? Dit kan je letterlijk uit de repository gegevens halen.

Er zijn 47 commits op verschillende dagen geplaatst, deze dagen komen vooral overeen met de lessen die we gehad hebben.

• Correctheid : valideert de code?

In de code voor de website van de klant zijn er geen validatie fouten. In zijn persoonlijke site heeft hij ook geen validatie fouten.

• Bereidheid tot verbetering: als je de commits bekijkt, zie je dan verbeteringen aan bestaande inhoud, of enkel nieuwe inhoud?

Meestal gebruikt hij nieuwe inhoud, maar ik zie af en toe ook is een verbetering van zijn commits die hij heeft geupload.
